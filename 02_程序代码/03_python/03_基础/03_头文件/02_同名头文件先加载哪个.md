如果你在 Python 环境里已经安装了 parlant，但主函数里又把本地 parlant 源码路径加进来了，最终调用哪个，取决于 sys.path 顺序。

规则是：

Python 导入时按 sys.path 从前往后找

先找到哪个 parlant，就用哪个

如果你写的是：

```python
sys.path.insert(0, r"D:\...\本地源码父目录")
import parlant
```

那通常会优先用本地源码

如果你只是 append 到后面：

```python
sys.path.append(r"D:\...\本地源码父目录")
import parlant
```

那大概率先用site-packages 里 pip 安装的版本

还有一个更关键的点：一旦本进程里已经 import 过 parlant，它会缓存到 sys.modules。这时候你后面再改 sys.path，通常也不会切换到另一个版本，除非你重启进程

最实用的判断方法是启动后打印：

```python
import parlant
print(parlant.__file__)
```

它会直接告诉你当前进程实际用的是哪一份 parlant
