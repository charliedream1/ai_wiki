这是 **Windows 系统默认路径长度限制（MAX_PATH=260 字符）** 导致的问题。Git 能提交是因为提交时只检查文件内容哈希，不检查本地路径长度；但拉取时需要在本地创建文件，Windows 默认不允许路径超过 260 字符，所以报错。

以下是 4 种解决方案，按推荐顺序尝试：

---

### 方法 1：配置 Git 允许长路径（最简单，优先尝试）
直接通过 Git 配置绕过 Windows 路径限制，这是最常用的解决方案。

1.  打开 **Git Bash** 或 **命令提示符（CMD）**（以管理员身份运行更佳）。
2.  执行以下命令：
    ```bash
    git config --global core.longpaths true
    ```
3.  重新执行拉取操作：
    ```bash
    git pull
    ```

---

### 方法 2：将仓库克隆到更短的根路径（临时快速解决）
如果不想改配置，可以通过缩短仓库的根路径来减少整体路径长度。

1.  将当前仓库移动到根目录下的短路径文件夹，例如：
    - 原路径：`C:\Users\你的用户名\Documents\long\long\long\path\repo`
    - 新路径：`C:\repo`
2.  在新路径下重新拉取：
    ```bash
    cd C:\repo
    git pull
    ```

---

### 方法 3：启用 Windows 系统级长路径支持（彻底解决）
Windows 10 1607 及以上版本支持通过系统设置彻底取消路径长度限制。

#### 步骤：
1.  按 `Win + R`，输入 `gpedit.msc` 打开**本地组策略编辑器**。
2.  依次展开：`计算机配置` → `管理模板` → `系统` → `文件系统`。
3.  找到并双击**启用 Win32 长路径**。
4.  选择**已启用**，点击「确定」。
5.  重启电脑（或至少重启 Git Bash/CMD），再重新拉取。

*如果是 Windows 家庭版（没有组策略编辑器），可以通过注册表修改：*
1.  按 `Win + R`，输入 `regedit` 打开注册表。
2.  定位到：`HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\FileSystem`。
3.  找到 `LongPathsEnabled`（如果没有就右键新建一个 `DWORD (32位) 值`，命名为 `LongPathsEnabled`）。
4.  双击将其值改为 `1`，点击「确定」。
5.  重启电脑后重试。

---

### 方法 4：重命名文件（从根源解决，需团队协作）
如果以上方法都不方便，或者需要跨平台协作（避免其他同事也遇到问题），可以和团队协商**缩短文件名或减少路径层级**。

1.  让能正常拉取的同事（比如用 Linux/macOS 的）在仓库中重命名该文件/文件夹，缩短名称。
2.  提交并推送重命名后的更改。
3.  你再重新拉取即可。

---

### 验证是否解决
执行完上述任一方法后，再次运行 `git pull`，如果没有报错且文件正常拉取，说明问题已解决。
