
# 🚨 C盘告急？8大隐藏空间杀手，一键清理释放30GB+！！

## 💬 1. 微信文件堆积（重灾区）

   - **📍 路径**：`C:\Users\yourName\Documents\WeChat Files\wxid_xxx\FileStorage\File`
   - **🧹 清理**：定期删除无用文件，或在微信设置中更改文件存储位置到其他盘符。
   - **⚠️ 注意**：删除前确认文件不再需要，已接收的文件删除后无法恢复；迁移存储位置后，历史文件需手动迁移。
   - **✨ 效果**：轻松释放 5-20GB 空间。

## 💻 2. IDE缓存爆炸（开发者必看）

   - **⚠️ 问题**：PyCharm/IntelliJ等IDE默认将配置、缓存、插件存在C盘，久用可达6GB+。
   - **🔧 解决**：
     - 进入 `帮助` -> `编辑自定义属性`，修改 `idea.system.path` 和 `idea.log.path` 到其他盘。
     - 使用 `帮助` -> `删除剩余的 IDE 目录` 清理旧版本残留。
   - **⚠️ 注意**：
     - 修改路径后需重启 IDE，首次启动会重新索引项目，可能较慢。
     - 删除旧版本目录前确认不再使用该版本，否则需重新安装。
     - 建议保留配置目录（config），仅迁移系统和缓存目录，避免丢失个性化设置。
   - **📖 参考**：[02_pycharm空间清理.md](./01_系统清理/02_pycharm空间清理.md)

## 🐍 3. pip缓存与Python包（程序员专属）

   - **⚡ 一键清理**：终端执行 `pip cache purge`。
   - **🔄 永久迁移**：设置环境变量 `PIP_CACHE_DIR` 指向其他盘（如 `D:\pip_cache`）。
   - **📦 Packages目录**：切勿直接删除 `C:\Users\用户名\AppData\Local\Packages`，可通过 `设置` -> `存储` -> `临时文件` 安全清理。
   - **⚠️ 注意**：
     - 清理 pip 缓存后，下次安装包时需重新下载，建议在网络良好时操作。
     - Packages 目录包含 Windows 应用商店应用数据，误删可能导致应用异常，务必通过系统工具清理。
   - **📖 参考**：[03_pip缓存清理.md](./01_系统清理/03_pip缓存清理.md)、[04_package目录清理.md](./01_系统清理/04_package目录清理.md)

## 🌐 4. Chrome浏览器缓存（隐形大户）

   - **📍 路径**：`C:\Users\用户名\AppData\Local\Google`
   - **👻 元凶**：Service Worker缓存、IndexedDB离线数据、AI模型文件（约4GB）。
   - **⚠️ 警告**：**绝对不能直接删除整个 Google 文件夹！** 会导致浏览器无法运行、丢失所有登录状态、书签和密码。
   - **🧹 安全清理方法**：
     - **方法一（推荐）**：快捷键 `Ctrl + Shift + Delete`，时间范围选"所有时间"，勾选"缓存的图片和文件"及"Cookie"。
     - **方法二（精准清理）**：地址栏输入 `chrome://settings/siteData`，按占用大小排序，删除占用大的网站数据。
     - **方法三（手动清理）**：关闭 Chrome 后，删除 `%LOCALAPPDATA%\Google\Chrome\User Data\Default\Service Worker` 和 `IndexedDB` 两个文件夹（会自动重建）。
   - **⚠️ 注意**：
     - 清除 Cookie 会导致所有网站退出登录，需重新输入账号密码。
     - 清除缓存后，常用网站首次加载会变慢，需重新下载资源。
     - 删除 IndexedDB 会清空网站的离线数据（如视频缓冲、本地保存的内容）。
   - **🚚 迁移方案**：将整个 `Google` 文件夹剪切到其他盘，使用管理员权限执行 `mklink /J` 创建符号链接，实现无损搬家。
   - **📖 参考**：[05_Google目录占用空间大.md](./01_系统清理/05_Google目录占用空间大.md)

## 🖥️ 5. 系统更新与休眠文件（系统级占用）

   - **🧹 磁盘清理**：使用 Windows 自带 `磁盘清理` 工具，勾选"Windows 更新清理"，可释放数GB。
   - **💤 休眠文件**：`hiberfil.sys` 占用 4-32GB。若无需快速启动，以管理员身份运行 `powercfg -h off` 关闭休眠。
   - **⚠️ 注意**：
     - 清理 Windows 更新缓存后，无法卸载已安装的更新补丁，若更新有问题需重新下载。
     - 关闭休眠功能后，"快速启动"选项也会失效，开机速度可能变慢。
     - 关闭休眠后将无法使用"休眠"模式（不同于睡眠），笔记本用户需注意电池续航影响。
   - **📖 参考**：[06_win自带清理工具.md](./01_系统清理/06_win自带清理工具.md)

## 🐾 6. VMware虚拟机缓存（虚拟化用户必看）

   - **⚠️ 问题**：拖拽/复制粘贴文件时，VMware会在C盘生成临时缓存且不主动清理，频繁传输可达10GB+。
   - **🎯 头号元凶**：拖拽缓存位于 `%localappdata%\Temp\vmware-%username%\VMwareDnD`
     - **🧹 清理**：直接删除该文件夹内所有内容，立即释放空间。
     - **💡 原理**：文件传输时会先在C盘做完整临时备份，传输完成后不主动清理，导致空间被"废弃的临时文件"占用。
     - **⚠️ 注意**：正在传输中的文件缓存不可删除，需等待传输完成后再清理。
   - **🎯 二号元凶**：挂起文件(.vmem)和日志文件(.log)
     - **🧹 清理**：在虚拟机存放目录搜索并删除不需要的 `.vmem` 和 `.log` 文件。
     - **🚚 迁移**：将整个虚拟机迁移到其他盘符，避免占用C盘。
     - **💡 说明**：挂起虚拟机会生成与内存大小相同的 `.vmem` 文件，长时间运行会产生大量日志。
     - **⚠️ 注意**：删除 `.vmem` 文件前需确保虚拟机已完全关闭，否则会丢失未保存的状态。
   - **💡 建议**：使用"共享文件夹"功能替代拖拽传输，速度快且无缓存问题。
   - **📖 参考**：[07_VMware占用空间.md](./01_系统清理/07_VMware占用空间.md)

## 🗑️ 7. Win11预装应用与遥测（一键瘦身）

   - **🛠️ 工具**：使用开源脚本 [Win11Debloat](https://github.com/Raphire/Win11Debloat)。
   - **⚙️ 操作**：管理员运行 PowerShell，执行 `&([scriptblock]::Create((irm "https://win11debloat.raphi.re/")))`。
   - **✨ 效果**：卸载无用预装应用、禁用遥测服务，提升性能并释放空间。
   - **⚠️ 注意**：
     - 卸载预装应用后，部分功能可能依赖这些应用（如照片查看器、邮件客户端），需提前安装替代软件。
     - 禁用遥测服务可能影响 Windows Update 和部分 Microsoft 服务的正常运行。
     - 建议在自定义模式下仔细选择要禁用的功能，避免误删必要组件。
     - 操作前建议创建系统还原点，以便出现问题时恢复。
   - **📖 参考**：[01_Win11Debloat.md](./01_系统清理/01_Win11Debloat.md)

## 💾 8. C盘扩容终极方案（当清理无效时）

   - **😫 痛点**：D盘有空间但"压缩卷"只能挤出少量空间，因"不可移动文件"阻挡。
   - **✅ 方案一（手动）**：将D盘大文件剪切到其他盘，腾出连续空间后再压缩。
   - **⭐ 方案二（推荐）**：使用 **傲梅分区助手** 或 **DiskGenius**，直接调整分区大小，自动处理文件排布。
   - **⚠️ 注意**：操作前务必备份重要数据！
   - **📖 参考**：[01_C盘无法扩容.md](./02_磁盘分区-扩容/01_C盘无法扩容.md)
