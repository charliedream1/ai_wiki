# 1. 通义灵码：
## 1.1. 介绍
  - 主页：https://lingma.aliyun.com/
  - 阿里开发，个人版目前免费
  - 支持skills/MCP/Specs开发模式
  - 提供基于VS Code的IDE / 同时提供Jetbrains的插件

## 1.2. skills包导入

方式1：手动创建

手动将目标 Skill 文件拷贝到下述路径后，重启 IDE，即可在对话框内通过 / 查看已加载的 Skills 列表。

| 位置   | 路径                                       | 作用域                 |
| ------ | ------------------------------------------ | ---------------------- |
| 用户级 | ~/.lingma/skills/{skill-name}/SKILL.md     | 当前用户的所有项目     |
| 项目级 | .lingma/skills/{skill-name}/SKILL.md       | 仅当前项目             |

**方式2：通过内置技能自动创建**

create-skill 是 Qoder 内置的技能创建助手。它通过交互式对话,引导你逐步创建符合规范的 SKILL.md 文件。使用方式:

```
/create-skill <技能描述,例如:将 Word 文档转换为 PDF>
```

如果你不熟悉 SKILL.md 文件的编写规范,建议优先使用 /create-skill 来生成初始模板,然后根据实际需求进行调整优化。

适用场景: 快速创建自定义技能,无需了解技能文件的详细格式。

如何使用

自动触发：直接描述需求，模型会自动判断是否使用合适的 Skill：
 
分析这个日志文件中的错误

模型将自动识别并调用 log-analyzer Skill。

手动触发：输入 /skill-name 手动触发：

``` 
/log-analyzer
```

# 2. Trae
## 2.1. 概要
  - 主页https://www.trae.cn/
  - 字节跳动开发，个人版目前免费，可使用顶级大模型
  - 支持skills/MCP
  - 提供基于VS Code的IDE
  - 对比下来，总体比通义灵码功能更强大，但是没有spec功能
  - 缺点：
    - 由于代码可能在沙箱中生成，导致python代码放在windows下报错提示有乱码，但实际又看不到
    - windows下会启用powershell，导致360杀毒报拦截错误

## 2.2. skills包导入

我们打开 Trae，点击右上角 ⚙ 按钮，再选择 规则与技能 菜单，在 技能 面板中选中 项目 （针对具体的项目设定技能），再点击 创建 按钮。 在弹出的对话框，点击 上传，选择下载好的 skills.md。 完成后，我们可以看到技能已经识别成功，可以开启或者关闭它： 对话时，当你明确知道需要使用某个技能时，你可以直接向 AI 发送指令，从而精准控制 AI 的输出。
