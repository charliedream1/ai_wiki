一个专门用来"PUA AI"的 Skill 插件 ( github上有将近15k的star )。

简单说，就是一套用大厂 PUA 话术驱动 AI Coding Agent 的 Skill 插件。你有没有遇到过这种情况：让 AI 调试一个 bug，它试了两下就说"我解决不了"然后撂挑子？OpenPUA 就是专门治这个毛病的。

它的核心逻辑是：AI 不允许轻易放弃。在彻底穷尽所有方案之前，不准说"做不到"。

怎么做到的？

插件内置了一套 L0 到 L4 的压力递增机制。第一次失败是温柔提醒，第二次失败开始质疑方案，第三次直接触发 L3 灵魂拷问，强制执行 7 项系统化清单——包括错误日志、环境检查、依赖项、替代方案、文档回溯等。到了 L4，就是最后通牒："其他模型都能解，你快毕业了。"

除了压力，它还内置了 5 步调试方法论：复现 → 隔离 → 假设 → 测试 → 验证，强制 AI 按顺序走完，不允许跳步。

作者用 9 个真实 bug 场景 × 18 组 A/B 对照实验跑了 benchmark，结果挺亮眼：修复数量提升 +36%，验证次数提升 +65%，工具调用提升 +50%，隐藏问题发现提升 +50%。

尤其是配置审查场景，没有 Skill 的组直接漏掉了 Redis 配置错误和 CORS 通配符安全风险，加了 Skill 之后全部揪出来了。
